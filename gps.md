# NMEA 0183 协议零基础入门

---

## GPS 模块是什么？

GPS 模块 = **天线 + 芯片**。它从天上的卫星收信号（很微弱的无线电波），算出自己在地球上的位置，然后通过**串口**把结果吐出来。

结果是一行一行的文字，像这样：

```
$GPRMC,123519,A,4807.038,N,01131.000,E,022.4,084.4,230394,003.1,W*6A
```

这就叫 **NMEA 0183** 协议。

---

## NMEA 0183 长什么样？

每条语句的固定格式：

```
$ -- 开头
  GP -- 系统标识 (GP=GPS, BD=北斗, GL=GLONASS, GA=Galileo)
    RMC -- 语句类型 (3个字母)
        ,数据,数据,数据... -- 逗号分隔的内容
                       *6A -- 校验和 (前面所有字符的 XOR)
```

> `$GP` 开头 = 美国的 GPS。
> `$BD` 开头 = 中国的北斗。
> `$GL` 开头 = 俄罗斯的格洛纳斯。

### 常见语句

| 语句 | 中文名 | 说什么 |
|------|--------|--------|
| **RMC** | 推荐最小导航信息 | 时间、位置、速度、航向 **(最关键的一条)**
| **GGA** | GPS 定位数据 | 海拔、卫星数、定位质量
| **GSA** | 精度因子 & 活跃卫星 | 哪些星参与了定位、定位精度(PDOP/HDOP/VDOP)
| **GSV** | 可见卫星列表 | 能看到哪些星、每颗的信号强度(SNR)
| **VTG** | 对地航向与速度 | 朝哪个方向走、走多快
| **GLL** | 经纬度定位 | 简洁版位置
| **ZDA** | 日期时间 | 精确到秒的 UTC 时间
| **TXT** | 文本消息 | 厂商信息、错误提示

## 逐句详解（写给小白）

### RMC — 最常用的一句

```
$GPRMC,123519,A,4807.038,N,01131.000,E,022.4,084.4,230394,003.1,W*6A
```

**手动拆解：**

| 字段 | 值 | 含义 |
|------|----|------|
| `$GPRMC` | | GPS 推荐最小导航信息 |
| `123519` | 12:35:19 | UTC 时间（北京时间 +8h = 20:35:19）|
| `A` | A=有效 V=无效 | 定位是否有效 |
| `4807.038,N` | 北纬 48°07.038' | 纬度 |
| `01131.000,E` | 东经 11°31.000' | 经度 |
| `022.4` | 22.4 节 | 地速（1 节 ≈ 1.85 km/h）|
| `084.4` | 84.4° | 航向角（0°=正北，顺时针）|
| `230394` | 1994-03-23 | 日期（年月日）|
| `003.1,W` | 西 3.1° | 磁偏角 |
| `*6A` | | 校验和 |

**怎么读懂位置？**

```
4807.038 = 48 度 + 07.038 分
         = 48 + 07.038/60
         = 48.1173°
```

- N = 北纬正数，S = 南纬负数
- E = 东经正数，W = 西经负数

### GGA — 定位质量

```
$GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,*47
```

| 字段 | 值 | 含义 |
|------|----|------|
| `1` | **定位质量** | 0=无定位, 1=GPS, 2=差分GPS, 4=RTK固定解 |
| `08` | **卫星数** | 参与了定位计算的卫星数量 |
| `0.9` | **HDOP** | 水平精度因子（越小越好，<1 优秀，>5 差）|
| `545.4,M` | 海拔 545.4 米 | 海拔高度 |

### GSA — 精度 & 参与定位的星

```
$GPGSA,A,3,02,05,08,12,14,18,22,26,30,,,,1.2,0.9,0.8*3F
```

| 字段 | 值 | 含义 |
|------|----|------|
| `A` | | 模式(A=自动, M=手动) |
| `3` | **定位模式** | 1=无定位, 2=2D(只有经纬度), 3=3D(有高度) |
| `02,05,08...` | **卫星 PRN 编号** | 哪些星参与了定位 |
| `1.2` | PDOP | 三维精度因子 |
| `0.9` | HDOP | 水平精度因子 |
| `0.8` | VDOP | 垂直精度因子 |

**DOP 值参考：**

| DOP | 等级 |
|-----|------|
| < 1 | 极好 |
| 1-2 | 优秀 |
| 2-5 | 良好 |
| 5-10 | 中等 |
| > 10 | 差 |

### GSV — 看到多少颗星？

```
$GPGSV,3,1,12,02,45,180,32,05,12,045,28,08,67,300,45,12,23,120,38*7A
$GPGSV,3,2,12,14,55,210,30,18,33,090,35,22,41,270,40,26,60,150,42*7B
$GPGSV,3,3,12,30,11,040,20,31,22,080,25,28,44,320,36,36,70,200,44*7C
```

| 字段 | 值 | 含义 |
|------|----|------|
| `3` | 共 3 条消息 | 卫星太多，一条装不下 |
| `1` | 第 1 条 | 这是第 1 条 |
| `12` | 共 12 颗 | 一共能看见 12 颗卫星 |
| `02` | PRN=2 | 卫星编号 |
| `45` | **仰角 45°** | 卫星在天空中多高（90°=头顶）|
| `180` | **方位角 180°** | 朝哪个方向（0°=正北，180°=正南）|
| `32` | **SNR 32dB-Hz** | 信号强度（越高越好，>30 良好，>40 极好）|

**怎么看懂卫星图？**
- 仰角高 → 卫星在头顶，信号好
- 仰角低 → 卫星在地平线附近，信号容易被遮挡
- SNR=0 → 搜到了这颗星但还没锁定

### VTG — 速度和方向

```
$GPVTG,084.4,T,081.5,M,022.4,N,041.5,K,A*XX
```

| 字段 | 值 | 含义 |
|------|----|------|
| `084.4,T` | 84.4° True | 真北航向（相对于地磁北极）|
| `081.5,M` | 81.5° Mag | 磁北航向 |
| `022.4,N` | 22.4 节 | 速度（节）|
| `041.5,K` | 41.5 km/h | 速度（公里/小时）|

### TXT — 模块说话

```
$GPTXT,01,01,02,u-blox ag - www.u-blox.com*50
$GPTXT,01,01,02,ANTSTATUS=OK*3B
```

u-blox 模块上电时会发 TXT 报告自己的硬件版本、固件版本、天线状态等。

## 关于坐标系

NMEA 用 **WGS-84** 坐标系（GPS 的默认坐标系）。

经纬度格式为 **度分制 (DMM)**：

```
4807.038 = 48°07.038'
         = 48 + 7.038/60
         = 48.1173°
```

**为什么不用小数点？** 因为 GPS 协议诞生于 1980 年代，度分制在纸面航海图上更容易手工标记，1 分 ≈ 1 海里。

---

## 关于校验和

NMEA 的校验和确保传输没有丢数据：

```
$GPRMC,123519,A,...*6A
               XOR ← 把 $ 和 * 之间的所有字符按位异或
                    6A = 计算结果，用 2 位十六进制表示
```

我们的库 `Parse()` 自动校验，校验不过会返回错误。

## 实战：判断定位好不好

| 指标 | 好 | 还行 | 差 |
|------|----|------|----|
| RMC.Status | `A` (Active) | | `V` (Void) |
| GGA.Quality | ≥ 1 | 0 = 无定位 |
| GSA.FixMode | 3 (3D) | 2 (2D) | 1 (无) |
| GSA.HDOP | < 2 | 2-5 | > 5 |
| GSV.SNR | > 40 | 30-40 | < 30 或 0 |
| GSV 可见星数 | ≥ 8 | 4-7 | < 4 |

## 名词速查

| 术语 | 含义 |
|------|------|
| **NMEA 0183** | GPS 设备的输出协议标准（1983 年发布）|
| **PRN** | 卫星编号 (02-32 = GPS, 120+ = 北斗/格洛纳斯) |
| **SNR** | 信噪比，衡量信号强弱的指标 |
| **HDOP/PDOP/VDOP** | 精度因子 (越小越好) |
| **UTC** | 协调世界时（北京时间 = UTC+8）|
| **WGS-84** | GPS 使用的全球坐标系 |
| **Baud Rate** | 串口波特率（GPS 默认 9600）|
| **2D/3D 定位** | 2D=只有经纬度（无高度），3D=有高度 |
| **冷启动** | 模块第一次开机或移动到很远，需要重新下载星历 |
| **热启动** | 模块之前定位过，快速找回位置 |
| **AGPS** | 通过网络辅助快速定位 |
